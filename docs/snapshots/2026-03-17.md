### Distribution strategy (dual deployment)
- **Cowork/Code:** Plugin distributed via org marketplace (GitHub-synced, admin-triggered sync)
- **Chat:** Standalone skills packaged as ZIPs with `cortex-` prefix, uploaded manually by admin via Claude Desktop (Organization settings > Skills)
- **Cortex MCP connectors:** Distributed separately as a custom connector URL — not part of this plugin

### Key decisions made
1. Plugin is the UX layer, not Cortex itself
2. Target audience: non-technical Dana Innovations employees on Claude Enterprise
3. Skills are the primary deliverable; plugin packaging is distribution infrastructure
4. Dual deployment needed because plugins don't work in Chat
5. Packaging script rewrites `name:` frontmatter to add `cortex-` prefix for Chat; source files stay unprefixed
6. CLAUDE.md stays under 100 lines as an index; detail goes in `docs/`
7. Develop on branches; both publish paths are admin-triggered (no accidental exposure)

### Platform constraints (verified March 2026)

**Surface capability matrix:**
| Capability | Chat | Cowork | Code |
|-----------|------|--------|------|
| Skills (standalone) | Yes | Yes | Yes |
| Skills (in plugin) | No | Yes | Yes |
| Plugins | No | Yes | Yes |
| Custom MCP connectors | Yes | Yes | Yes |
| Hooks | No | No | Yes |
| Sub-agents | No | Yes | Yes |
| Local file access | No | Yes (VM, bridged) | Yes (direct) |

**Execution environments:**
| Surface | Environment | Local files | Terminal |
|---------|------------|-------------|----------|
| Code | Raw terminal | Full, direct | Unrestricted |
| Cowork | Local VM | Yes, with permission (isolated but bridged) | Via VM |
| Chat | Anthropic cloud sandbox | No | Sandboxed only |

**Hard constraints:**
- Skills do not sync across surfaces
- `/v1/skills` API does NOT provision to claude.ai
- MCP prompts do NOT surface in Chat UI
- Admin skill provisioning for Chat = manual ZIP upload only
- Plugin marketplace GitHub sync = admin-triggered, not automatic

### Dev/testing workflow
